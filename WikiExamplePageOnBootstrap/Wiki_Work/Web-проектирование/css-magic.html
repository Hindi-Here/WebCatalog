CSS магия

  CSS предлагает широкие возможности для создания интерактивных страниц. Анимации в CSS являются удобным средством для разработки динамического и привлекательного сайта с различными эффектами, улучшающими пользовательский опыт. CSS-анимации позволяют добавлять реакции (подсветка, слои, клики, анимация, и другое) на страницу без использования JS, что также влияет на производительность (анимации на JS будут более требовательны к оптимизации). Анимации, обычно, сглаживают переходы между страницы и состоянием элементом, тем самым создавая эффект общности (целостности всех элементов с отсутствием резких переходов).

  К полезным инструментам, используемых часто в анимациях, можно отнести filter и transform. Первый накладывает различные визуальные эффекты к элементам, например, размытие (blur), контрастность (contrast), яркость (brightness), прозрачность (opacity), инвертирование (invert), преобразование в серый (grayscale), сепия (sepia), и другие эффекты. Синтаксис всех этих значений прост, в скобках в качестве параметра указывается процентное соотношение или в десятичном формате (0.3, 1.2, и т.п.). Transform же предназначен для геометрических трансформаций. Например, translateX и translateY используются для перемещения объекта в пространстве, что может быть полезно в анимациях передвижения. Scale увеличивает или уменьшает размер объекта (стандартной величиной является 1 или 100%). Rotate позволяет вращать элемент вокруг своей оси, а в качестве параметра принимается deg единица (градусы).

  И filter, и transform, поддерживают несколько глобальных значений. «filter: none» удаляет все фильтры, которые применялись ранее. Inherit наследует фильтры от родителя, а unset в зависимости от того наследуется свойство или нет, устанавливается в значение inherit или initial (установка исходного значения).

  Анимацию в CSS можно осуществить двумя способами: keyframes и транзитивная анимация. Особенность подхода через ключи заключается в создании более гибких и сложных анимаций. В @keyframes указывается имя анимации, а в теле самой анимации указываются ключи, которые задаются ключевыми словами to (0%) и from (100%). В ситуациях, когда необходимо определить больше ключей используется числовое именование (то есть: 0%, 25%, 50%...). На рисунке 28 показан пример анимации переливания градиента на странице с помощью keyframes подхода.

  В body указываются нулевые отступы, чтобы убрать любые стандартные отступы, устанавливаемые браузерами. В качестве изображения (background-image) ставится значение linear-gradient (линейный градиент) с направлением слева направо и перечнем переливающихся цветов. Для того, чтобы осуществить анимацию необходимо увеличить размерность градиента, что осуществляется в background-size (1000% – увеличение по ширине, 100% по высоте). Сама анимация имеет имя gradient и состоит из трех ключей (0%, 50%, 100%), где на каждом фрагменте изменяется позиция фона страницы, что создает эффект переливания цветов.

  Animation включает в себя несколько параметров для заполнения (не все они строго обязательны для указания): animation-name (имя анимации), animation-duration (продолжительность), animation-timing-function (скоростная кривая анимации), animation-delay (задержка перед началом анимации), animation-iteration-count (количество повторений), animation-direction (направление анимации), animation-fill-mode (стиль, применяемый к элементу после выполнения анимации), animation-play-state (состояние воспроизведения или паузы).

  Объявлять анимацию можно как составным образом (то есть указывать каждый параметр отдельно), так и через сокращение animation (порядок объявления такой же, как и порядок описания в прошлом абзаце, при этом можно пропускать некоторые записи без нарушения синтаксиса).   Рассмотрим пример с анимацией игрового поля крестиков ноликов. Сама HTML структура достаточно проста: имеется главный контейнер с настроенной сеткой 3 на 3 с размерами 100pt, а также дочерние ячейки, заполняющие эту сетку. Каждая ячейка имеет свой уникальный id (cell1, cell2…). Задача состоит в создании анимации, которая пошагово будет отображать появление элементов с разных ракурсов (верхние ячейки появляются сверху вниз, правые движутся слева направо, и так далее). Для анимации по всем направлениям создается 4 анимации с наименованиями animation-направление-cell и animation-middle-cell для центральной ячейки, которая используется scale.

  На рисунке 29 показан пример появления верхнего ряда. В cell1 показано объявление каждого параметра отдельно, а в остальных примерах – краткая форма записи анимации. Аргументы идут следующим образом друг за другом: имя анимации, время действия, задержка, стиль анимации после завершения (хороший пример того, как были пропущены аргументы animation-timing-function перед animation-delay, а также animation-iteration-count и animation-direction перед animation-fill-mode без нарушения синтаксиса). Задержка анимации с приростом на 0.3 секунды для каждой ячейки позволяет осуществить последовательное выполнение анимации. Forward сохраняет стиль согласно последнему кадру анимации (фиксирует объект после его анимации на месте, где ячейка закончила движение).

  В keyframes animation-top-cell используется два ключа: начало анимации (0%) и конец (100%). В начале анимации объекты находятся чуть выше своего положения, указанного в CSS стиле, и имеют нулевую видимость. По концу же анимации объекты становятся видимыми и возвращаются в начальное положение. Например, если элемент находится на позиции 100px по высоте, то в начале анимации его позиция уже будет на n пикселей выше начальной, а к концу анимации объект вернется на высоту 100px. Таким образом создается эффект плавного появления и передвижения элементов сверху вниз. В качестве параметра для translateY можно указывать не только проценты, но и другие единицы (px, pt, и так далее).

  Animation-name принимает лишь один параметр – имя анимации. При указании длительности (animation-duration) и задержки (animation-delay) указывается числовое значение, выраженное в секундах. Количество итераций (animation-iteration-count) может иметь одно из двух значений: число повторений, infinity (бесконечное проигрывание). Направление анимации (animation-direction) не может иметь пользовательских значений, вместо этого выбирается один из следующих вариантов направления: normal (от начала до конца анимации), reverse (от конца к началу), alternate (сначала в прямом, затем в обратном порядке), alternate-reverse (инверсия версии alternate). Animation-play-state определяет воспроизводится (running) анимация или стоит на паузе (pause).

  Транзитивная анимация позволяет плавно изменять стили, переключаясь между двумя состояниями элемента. Обычно, транзитивная анимация применяется совместно с событием наведения курсора или при работе с другими псевдоклассами. Синтаксис transition включает: изменяемое свойство, продолжительность анимации, функция, задержка. На рисунке 30 показан пример применения транзитивной анимации для плавной анимации подсветки ячеек и появления текста при наведении и кликах.

  В TicTatToe-cell устанавливаются стили для дочерних элементов (блоки для игры в сетке 3 на 3). Прозрачность ячеек устанавливается на 0, что задействуется в анимации из рисунка 29. Color также устанавливается прозрачным, чтобы появление текста после клика на ячейки было менее резким. В transition устанавливается обработка двух свойств, разделенных запятой (таким образом, анимация охватывает и задний фон, и цвет текста). В качестве второго параметра указывается длительность анимации и функция кривой.

  Псевдоклассы в CSS предназначены для стилизации элементов в зависимости от их состояния. Ярким примеров является hover, который определяет стилизацию при наведении компьютерной мышью на элемент. Но существует еще множество другим псевдоклассов, которые можно использовать для стилизации элементов. Например, active активируется в момент клика на элемент. То есть, при нажатии на элемент активируется событие, описанное в псевдоклассе active, и продолжается до тех пор, пока пользователь не отпустит зажатую кнопку мыши. Псевдокласс focus определяет стиль во время активности элемента. Обычно, focus используется совместно с полями ввода input (событие срабатывает, когда пользователь кликает на это поле) или же при переходе через табуляцию (если элемент помечен через tabindex, то focus также срабатывает на этом элементе).

  Говоря о формах, существует ряд псевдоклассов, которые часто употребляются вместе с ними. Например, disabled и enabled для стилизации элементов во включенном и отключенном состоянии. Или required и optional, определяющие обязательность полей в заполняемой форме. Если у вас существует проверка на корректность введенных данных, то может пригодится valid и invalid, которые активируются, когда форма валидна или не валидна. Чтобы корректно работали эти псевдоклассы, элементы должны иметь соответствующие атрибуты по типу type для валидации или required для подчеркивания обязательности.

  Для стилизации ссылок используют link и visited псевдоклассы. Первый срабатывает, если ссылка не была посещена, а второй в иной ситуации. Если на форме расположены checkbox элементы, то может быть уместным использование checked, срабатывающий, если элемент активирован.

  Если у вас существует несколько одинаковых дочерних элементов без их id, то для стилизации конкретной единицы применимы псевдоклассы: nth-child, nth-last-child, nth-of-type, nth-last-of-type. Например, при использовании «TicTacToe-cell:nth-last-child(2)» стилизован будет второй элемент с конца (если вернуться к примеру с крестиками ноликами, то это 8-ая ячейка). Аналог без last суффикса означает отчет с начала элементов, находящихся внутри родителя. Если необходимо учитывать тип элемента, то тогда используется псевдоклассы nth-of-type или nth-last-of-type.

  Имеется некая форма для регистрации на онлайн конференцию, состоящего из полей: ФИО, почта, номер телефона, компания, должность, пароль. Некоторые из этих полей имеют валидацию данных, которая происходит через type с указанием типа вводимых данных, а также с помощью паттернов. Регулярные выражения в этих паттернах регулируют допустимые символы и их длину (это также можно сделать через атрибуты minLength и maxLength). На рисунке 32 показана структура формы регистрации.

  Паттерн работает на основе регулярного выражения, которое строится из специальных символов. Например, паттерн «[a-zA-Zа-яА-Я ]{8,100}» говорит о том, что допускаются русские и англоязычные символы разных регистров (малые и заглавные символы), а также пробелы. Длительность последовательности должна содержать минимум 8 символов и максимум 100, что зафиксировано в фигурных скобках в конце выражения. Placeholder является атрибутом, описывающий поле (если поле пустое и без фокуса, то будет видно описание placeholder), что необходимо для пояснения формата записи паттерна. Type email проверяет на соответствие почте, а type password скрывает вводимые символы, чтобы скрыть пароль.

  Для отслеживания правильности ввода можно воспользоваться псевдоклассами valid и invalid. На рисунке 33 показано, как они применяются к обязательным полям с атрибутом required. Подобным образом эти псевдоклассы применимы с необязательными полями через optional или ко всем без учета дополнительной фильтрации. К элементам можно обращаться через родительский элемент. Например, «form-group input» обращается во всем input элемента, находящимся внутри родительского класса form-group.

  Таким образом, при соблюденном паттерне в обязательных полях – границы будут подсвечены зеленым оттенком. В противном случае – останутся красными. Остальные (необязательные поля) имеют светло-синий оттенок, обозначающий, что поля можно пропустить при регистрации, поскольку не все регистрирующиеся могут являться частью компании и работать.   С помощью комбинации CSS + JS можно создать уникальные анимации, которые запомнятся пользователю. Вы можете создавать эффекты параллакса и другие сложные эффекты, создавать события для компьютерной мыши по типу испускания дыма при движении, настраивать поведение полосы прокрутки, и многое другое.
