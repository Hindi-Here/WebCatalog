Hello world в web-проектировании

  В web-проектировании начальной темой изучения является язык разметки (HTML), язык стилей (CSS), а также язык сценариев (JS). Конечно, в некоторых случаях обучение верстке сайтов может отличаться, например, когда разработчик изначально изучает конструкторы сайтов. Под конструкторами понимается инструмент, который предоставляет готовые блоки, которые самостоятельно генерируют программный код. Достаточно переместить желаемый элемент на специальное поле, являющиеся отражение web-страницы, а затем настроить его свойства через отдельную панель. Примерами таких конструкторов являются CMS системы (WordPress или Joomla), онлайн конструкторы (генерируют сайт по шаблону).

  HTML представляет собой набор тегов, из которых строятся блоки сайта и их содержимое. В самой простой вариации web-сайт имеет вид, как показано на рисунке 1.

  В обязательном порядке используются теги: html, head, body. Html обозначает саму структуру, поэтому употребляется тег только в начале и конце всей структуры и нигде более. В head указываются ссылки на внешние файлы (это могут быть скрипты, стили, библиотеки), для чего используется link внутри head структуры. Также в заголовок (так часто называют head блок) включены meta теги и наименование ресурса в title. Тег meta играет важную роль и определяет метаданные, предназначенные для браузеров и поисковых систем. Через атрибут charset определяется кодировка документа, а с помощью сочетания name (доступны author, description, keywords, viewport значения) и content задается информация об авторе страницы, ее описания и ключевых слов, относящихся к содержимому, управление размером окна и его масштабом. В теле (body) размещаются все остальные элементы (кнопки, текстовые поля, текст, изображения, и другой контент).

  На рисунке 1 показано несколько семантических тегов (это теги, которые своим названием дают понять о содержимом), которые предназначены для улучшения читаемости структуры и доступности web-страниц. Обычно сайт разбивают на шапку (header), основное содержимое (main), подвал (footer). Достаточно часто используют теги nav для обозначения навигационного меню, section в качестве раздела и article, как самостоятельную единицу от секции (например, одна секция содержит несколько артиклей, являющихся отдельной новостной статьей).

  Теги включают в себя атрибуты, предоставляющие дополнительную информацию об элементах и помогающие управлять их поведением и форматированием. Атрибуты представляют собой пары «ключ-значение». Например, на рисунке 1 показан маркированный список (ul и li), включающий ссылку (a) с атрибутом href, который хранит ссылку на источник. Таким образом, при клике на «Панель 1» или «Панель 2» пользователя перенесет на другой сайт по ссылке «https://…». Часто при переходе по ссылкам использует дополнительно атрибут target для управления переходом. При установке «target = _blank» документ будет открываться в новой вкладке браузера (по умолчанию переход происходит в той же вкладке). У target есть и другие вариации, такие, как _self и _parent, открывающие ссылку в текущем блоке или блоке родителе (используется в теге iframe).

  На этом же рисунке в теге изображения (img) указано три атрибута: ширина (width), высота (height), ссылка на рисунок (src). Ссылка на изображение для web-страницы обычно берется, исходя из корневой папки проекта, или напрямую из интернета. Однако, большинство атрибутов не указываются непосредственно внутри тега, но описываются в стилях. Для работы со стилями элементы идентифицируются через class или id.

  Стили можно прописывать в отдельном теге style, который указывается в заголовочной части. Или подключаются через link в случаях, когда существует отдельный стилевой файл. На рисунке 2 показана стилизация элементов web-страницы.

  Предположим стоит задача составить простой онлайн калькулятор площади трапеции (по базовой формуле). В качестве основного блока указан div тег с классом figureCalculateBlock, заголовок блока в виде p (текстовый тег) с классом figureCalculateTitle. Аналогичным образом устанавливается класс для изображения и кнопки на странице.

  Для обращения к элементам по их классу используется «.», а в случаях с id – «#». Вы можете задать стили для всех компонентов одного тега, как показано на примере с input. Таким образом, все объекты текстовых полей будут иметь внешние отступы «10, 0, 10, 0» (сверху, слева, снизу, справа), внутренний отступ со всех сторон в 5 пикселей (padding свойство), скругленные границы черного цвета с жирностью в 2 пикселя. Width 100% в этом примере растягивает поле по длине блока (если input будет вне div или другого тега, то растянется по ширине экрана).

  Часто могут возникать ситуации, когда есть несколько элементов имеют одинаковый стиль (или с незначительными отличиями). В таком случае рекомендуется обращаться к элементам через запятую, например, «.figureCalculateTitle, .figureCalculateDescription».

  Блок (div) figureCalculateBlock имеет ширину в 300 пикселей и высотой с размером, устанавливаемым в зависимости от количества контента внутри блока. Внутренний padding отступ позволяет отдалить элементы от краев блока, что улучшает визуальную компоновку. Затем настраиваются границы устанавливается: сплошная линия с серым оттенком (border), округление всех углов на 15 пикселей (border-radius), тень блока для эффекта блочности (box-shadow). Display в figureCalculateBlock указывает на то, как будут отображаться объекты на странице. Значение flex означает, что элементы будут идти друг за другом, а flex-direction устанавливает вектор последовательности (column означает отображение сверху вниз). На рисунке 3 показан результат отображения элементов внутри блока через flex.

  В заголовке figureCalculateTitle для текста задается размерность в 24 пикселя (font-size), жирность (font-weight), нижний внешний отступ (margin с 4 параметрами можно заменить на margin-bottom в этом случае). Помимо показанных свойств, для текста в CSS существует множество других параметров, которые часто используются верстальщиками. Например, шрифт (font-family), стиль (font-style с параметрами курсива и наклона), декорации (text-decoration с возможностью подчеркивания под/над/внутри текста и разной стилистикой линий: пунктирной, двойной, сплошной, и так далее), тень (text-shadow, идентичный принцип применения с box-shadow), выравнивание (text-align/vertical-align, позволяющий выравнивать контент по: левому краю, по правому краю, ширине, центру), направление (direction с отображением текста слева направо или справа налево), пробелы между символами (letter-spacing/word-spacing/white-spacing), переполнение текста (text-overflow, редактирующий поведение текста при выходе за пределы пространства).

  Input устанавливает стиль для всех полей ввода на web-странице. Эти элементы также имеют скругленный радиус, внутренний и внешний отступы. Input растянут по всей длине блока с учетом padding в 20px в figureCalculateBlock. Если элемент, к которому вы применяется width 100% находится внутри какого-то тега, то он будет выравниваться именно по родительскому элементу. Например, если вы задали width 100% для блока, который внутри body и не заключен внутри других тегов, то ваш блок будет распространяться на ширину всей страницы.

  Для кнопки figureCalculateButton задается зеленый фон (background-color) с белым текстом (color). Линии границ при этом невидимы, но очерчены фоном кнопки. При наведении на кнопку курсор меняет свой внешний вид на pointer, что придает больше видимости кликабельности по кнопке. Псевдокласс hover используется для стилизации кнопки при наведении. Для применения события наведения используется «:hover» в добавок к имени элемента. Таким образом, при наведении на кнопку, ее цвет будет затемняться.

  Полученный результат рассмотренного CSS кода можно рассмотреть на рисунке 3 (для улучшения представления рекомендуется добавить верхний отступ для кнопки и нижний для изображения).

  Однако, на данный момент, какие бы значения не вводились – результат не будет выдаваться. Для создания сценария подсчета площади необходимо подключить script, что можно осуществить через тег с соответствующим именем (в таком случае рекомендуется указывать его в конце body для оптимизации загрузки страницы) или подключить в заголовке (head) через атрибут src.

  В JS нет строгой типизации, поэтому переменная может быть одновременно, как строкой, так и числом или массивом. В качестве переменной используется ключевые слова let, const, var. Var является глобальной переменной, а в современном коде из-за его не рекомендуется использоваться (глобальные переменные могут вызывать путаницу из-за общего доступа из любого места программы). Let применяются к элементам, которые могут изменяться в коде, а const является константой, которая неизменяемая (однако, в массиве свойство может быть изменено).

  Для получения данных из Input на странице используется конструкция document.GetElement, которая вытягивает нужные данные по их атрибуту. У GetElement есть разные вариации, позволяющие получать данные по Id, Class, Attribute, Tags, Selector, и другие. Каждому текстовому полю соответствует переменная, которая будет участвовать в дальнейших вычислениях (смотрите let area на рисунке 4).

  Можно установить дополнительные проверки на нулевые значения (пустые поля), а также на положительность числа, поскольку сторона трапеции не может быть отрицательной. В Input можно установить «type = number», чтобы был доступен ввод только числовых значений.

  Всплывающее модальное окно (alert) редко используется на практике. Вместо него через JS код происходит редактирование поля для ответа, либо генерируется новый HTML элемент. В некоторых ситуациях может создаваться кастомное окно, появляющееся на экране, однако, это также не частая практика.

  Предположим, что у нас имеется пустой элемент span с именем answer в figureCalculateBlock, расположенный под кнопкой figureCalculateButton. Для того, чтобы динамически изменять содержимое данного элемента используется свойство innerText, применяемый к QuerySelector(‘.answer’). Таким образом, при клике на кнопку – форма слегка расширится вниз с текстом, определенным разработчиком в аргументах innerText. Для создания новых элементов используется createElement, который принимает в качестве параметра тег элемента. Для придачи ему свойств и стилизации поддерживается style с перечнем атрибутов. На рисунке 5 показано несколько вариаций создания. Для формирования в нужном месте определяем родителя (parent), в котором будет располагаться создаваемый тег. По итогу получаем текст с отступом сверху в 10 пикселей, жирного синеватого оттенка.

  Append добавляет элемент в конец, а аналогичный метод prepend в начало списка элементов родителя. Для точечной вставки можно воспользоваться insertAdjacentText, insertAdjacentHTML, insertAdjacentElement для вставки HTML кода, текста, элемента. Вы можете заменять один элемент другим через replaceWith, который заменяет текущий элемент на новый.

  Одной из проблем такой генерации может быть неконтролируемое появление элементов, когда при срабатывании одной и той же функции, создаются элементы и позиционируются друг за другом. В этом случае лучше воспользоваться пустым изменяемым элементом из HTML кода или предварительным удалением через remove/removeChild и других методов удаления.

  Существует множество способов нахождения площади одной и той же фигуры. Например, для трапеции базовой формулой является через основания и высоту «(a+b) / 2 * h», но также в определенных ситуациях площадь может находится через диагонали и угол «(d1 * d2) / 2 * sin(x)», через четыре стороны (сложная формула), по средней линии и высоте (m * h), и другие формулы. Естественно, будет неудобно формировать множество схожих блоков для нахождения по каждой формуле отдельно.

  Для решения этой проблемы добавим select элемент на web-страницу. Select – это выпадающий список, позиции которого задаются через option. На рисунке 6 показан обновленный HTML блок по задаче на нахождение площади трапеции.

  Разберем на примере трех основных формул, но принцип работы с большим количеством будет крайне схож. Придаем select имя calcMethod, который будет стилизован таким же образом, как и Input элементы. Добавляем, помимо этого, событие onchange, которое отслеживает смену позиции (то есть, при выборе нового варианта будет запускать функция generateTrapezoidFields в JS коде). Внутри select задаются наименования позиций в тегах option с атрибутом value. Value – можно охарактеризовать именем опции, к которому будет идти обращение в скрипте.

  Для Input вместо fieldA, fieldB, fieldH устанавливаем нумерацию, поскольку записанные данные могут иметь разное значение в зависимости от выбранного метода подсчета (грубо говоря, это необходимо для улучшения читаемости кода).

  Предполагается, что программа будет работать следующим образом: пользователь выбирает способ подсчета площади в выпадающем списке (по умолчанию используется формат «(a+b) / 2 * h»). Затем, после ввода данных, при нажатии на кнопку «Подсчитать» происходит подсчет площади по определенному закону. Главная задача calculateTrapezoidArea считать value из select списка и запустить соответствующий метод. Это достигается через switch case, внешний вид которого показан на рисунке 7.

  В кейсах запускается один из сценариев: standartArea, diagonalsArea, middleLineArea (на рисунке он не показан, но содержимое такое же). В этих функциях изначально считываются все участвующие в вычислениях полях через getElementById (чтобы избежать повторений, как показано на рисунке, вы можете считывать эти данные в отдельной функции, а затем передавать значения в функции вычисления площади). В новой переменной area происходит подсчет суммы, а сам результат передается позже в элемент answer (id последнего span элемента). ToFixed, применяемая к area, фиксирует то, сколько максимально знаков после запятой.

  Проблема, возникающая после применения этого алгоритма, заключается в том, что при смене сценария подсчета площади описание полей placeholder остается неизменным, а при выборе подсчета по центральной линии и высоте остается лишнее третье поле. Для решения этой проблемы также используется switch оператор, в котором настраиваются стили всех полей. На рисунке 8 показан алгоритм изменения свойств каждого поля (их описание и видимость) при срабатывании одного из трех сценариев подсчета. Также, при смене позиции в select очищаются все ранее заполненные поля (в том числе и answer).

  Таким образом, при смене значения выпадающего списка будут изменяться комментарии внутри input в зависимости от содержимого case, а также стираться видимость третьего поля при срабатывании middleLine. Однако, показанный на рисунке 8 алгоритм можно улучшить и оптимизировать, применяя стили элементов через контейнеры (то есть осуществлять переборку контейнера со свойствами полей). Это может быть значительно удобней, нежели использование дублирования кода для каждого сценария (особенно, если полей ввода еще больше, чем три).
