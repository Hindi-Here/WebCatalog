Клиентская и серверная сторона

  При проектировании web-проекта важно учитывать клиентскую (клиент) и серверную сторону (сервер).

  Клиент – часть проекта, которая принимает данные от сервера и отображает их пользователю. Под клиентом понимается интерфейс, который виден пользователю. Например, web-страницы в браузере или приложения на мобильном устройстве. Все, что видит пользователь и то, с чем он может взаимодействовать можно отнести к клиентской стороне. В рамках web-проектирования, к клиенту можно отнести: отображение контента на странице, обработка сценариев клика (и других событий), анимации компонентов, валидацию данных, обработку заявок в формах, исполнение клиентской логики без участия сервера. Клиентская часть формирует запрос к серверу, а затем обрабатывает полученный ответ. Реализуется клиент с помощью HTML, CSS, JS (и их фреймворков по типу Ajax, React и Vue).

  Сервер – часть проекта, которая принимает HTTP-запросы от клиентов, а затем передает ответ, который отображается клиентом пользователю. Серверная часть невидима для пользователя. Например, взаимодействие с базой данных, реализация логики для обработки данных и их представления, динамическое создание страницы, обеспечение защиты, управление сессиями, отправка уведомлений, управление ресурсами. Серверная часть реализуется через языки программирования (PHP, Python, Ruby, C#, и так далее).

  Некоторые web-сайты включают только сторону клиента. К таким страницам можно отнести статические web-сайты, где все взаимодействия происходят только на клиенте. К этой же категории относятся одностраничные сайты, офлайн приложения, интерактивные проекты (подразумеваются показательные проекты по типу презентаций или демонстраций продукта). Однако, все большие проекты (а они по своей сути все коммерческие) включают в себя серверную логику, поскольку сервер позволяет более точечно работать с пользователями (персонализация услуг, сбор данных, рассылка, геймификация, и прочие взаимодействия).

  Подводя небольшие итоги, клиент является видимой пользователем частью, которая формируется из языков разметки, стилей, сценариев. Сервер является невидимой частью и представляет собой внутреннюю логику, построенную на языках программирования. Взаимодействие между этими сторонами происходит через HTTP-запрос. Клиент отправляет запрос на сервер, а сервер обрабатывает этот запрос и выдает HTTP-ответ клиенту. После получения ответа, клиент отображает полученный ответ в виде измененного интерфейса.
