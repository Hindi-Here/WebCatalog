Frontent фреймворк Bootstrap

  Bootstrap – frontend фреймворк, предназначенный для быстрого создания адаптивного дизайна web-сайтов и приложений. Он состоит из файлов создания и растягивания сетки, стилизации контента, вспомогательных служебных классов (выравнивание текста, настройки цветов, и так далее). К преимуществам этого фреймворка можно отнести легкость использования из-за наличия уже готовых шаблонов и адаптивности, что делает его хорошим решениям для новичков. Однако, из-за наличия единой стилизации всех элементов, bootstrap нет так полезен при создании уникального дизайна.

  Официальная документация включает в себя описание самого фреймворка, а также предоставляет множество готовых шаблонов UI дизайна.

  Все элементы заключены в блок с классом dropdown. Этот класс вы можете свободно переименовать или убрать, если это необходимо (dropdown в этом примере просто является маркером). Однако, не рекомендуется осуществлять какие-то манипуляции в конструкциях Bootstrap (их переименование), поскольку их изменение нарушит стилизацию элементов. Тем не менее, вы можете добавлять свои стили к уже существующим или прописывать свое имя элемента, добавив его в начале цепочки свойств.

  Часть btn в начале класса указывается на тип элемента (кнопка), а btn-secondary на ее внешний вид. Dropdown-toggle добавляет к тексту «Dropdown button» иконку стрелки, смотрящей вниз для обозначения содержания списка (без dropdown-toggle также можно обойтись, если вам не нужна эта иконка). Атрибут data-bs-toggle указывает, что элемент должен открывать меню выпадающего списка. Классы, указанные внутри списка необходимы для стилизации меню. Чтобы изменить их стиль, то можно просто в CSS файле прописать нужные параметры для изменения

  В Bootstrap существует своя цветовая схема, описанная в root (корень документа). Для того, чтобы воспользоваться настроенными цветами используется метод var в качестве значения атрибутов. На рисунке ниже показана урезанная версия цветовой схемы фреймворка и ее использование для элементов (все значения цветов во вкладке CSS variables). Для некоторых элементом можно указать режим цветовой палитры (светлая и темная тема). Например, для кнопок или выпадающих списков можно использовать атрибут data-bs-theme, принимающий параметры light (светлая) и dark (темная тема).

  Существует ряд заготовленных тем (dark, success, warning, и другие темы во вкладке color официальной документации), которые можно использовать для стилизации некоторых элементов таких, как кнопки, формы, таблицы. Подобная стилизация уже применялась на рисунке 35 для кнопки (btn-secondary). При смене значения на btn-success мы получим зеленый стиль, а primary отражает синие оттенки (другие темы, соответственно, дают иные цвета). Вы можете использовать btn-outline для указания темы границ. То есть, при указании btn-outline-success граница и текст кнопки будут зелеными, а background белым.

  В разделе Layout показаны способы компоновки элементов и формирование сетки. Сетка в Bootstrap формируется за счет блоков с классом container, row (строка сетки), col (столбец сетки). Grid состоит из 12 столбцов, что должно учитываться при размещении элементов. Если необходимо формировать два столбца, то col для каждого столбца имеет вид «col-6» (12/6 = 2). Таким образом можно указывать ширину контента внутри родительской строки (row класс). На рисунке ниже будет показана компоновка шапки, основного контента, подвала. Например, шапка включает в себя три столбца с параметрами: col-8, col-2, col-2. Это означает, что первый столбце будет составлять 2/3 от ширины экрана, а остальные два столбца 1/3.

  Важной частью адаптивности контейнеров такого рода являются breakpoint. Эти точки определяют момент, когда макет может видоизменяться. Выделяются следующие точки останова (breakpoints): small (sm > 576px), medium (md > 768px), large (lg > 992px), extra large (xl > 1200px), extra extra large (xxl > 1400px). Эти точки работают по следующему принципу: если ширина экрана больше значения, соответствующее breakpoint, то действует модель макет, закрепленное за этой точкой. Если же, экран пользователя меньше указанного, то задействуется новая модель. На рисунке 38 при описании последней строки сетки показан принцип смена макетов. Если размер экрана больше xl (1200px), то в строке содержится три столбца, но, если размер меньше 1200, то задействуется макет md, в котором осуществлен переход последней записи на новую строку, поскольку размер колонн становится 6/12 ширины контейнера.

  У класса container также имеются breakpoints. Точки в контейнере работают немного по-другому. Они определяет размерность контейнера в двух состояниях: 100% и фиксированный размер. Например, если блок имеет container-lg класс, то сам блок будет фиксированного размера до тех пор, пока ширина экрана не станет меньше 768 пикселей. До этого момент ширина блока фиксировано может быть: 960px, 1140px, 1320px (в зависимости от других точек, во вкладке Container в документации есть полезная таблица на эту тему). Когда размер экрана меньше 768, то блок растягивается на весь экран (100%). Так работают все контейнеры с точками останова (помимо fluid – 100% растягивание при любом экране).

  Если вы не можете найти нужного свойства в Bootstrap, или применение разных конструкций приводит к конфликту, то можно описать некоторые стили в CSS, обращаясь к элементам по их классу или id. Например, для строк макета был установлен текст с размером 24 пикселя, видимые границы и сероватый оттенок в качестве заднего фона. А к самой сетке пунктирный border синеватого оттенка из primary темы. Результат стилизации и форматирования сетки показан на рисунке 39.

  Можно сказать, что получилась первоначальная разметка, характеризующая структуру сайта. В header панель функционала сайта, блок поиска и регистрации. В теле сам контент и навигация по этому контенту. А в подвале контактные данные.

  При позиционировании объектов часто нужны внешние и внутренние отступы. Для таких отступов в Bootstrap используются p и m записи с диапазоном значение от 0 до 5 (по умолчанию 0rem, 0.25rem, 0.5rem, 1rem, 1.5rem, 3rem). Для задачи направления используются сокращения t, b, s, e (top, bottom, start, end), а также x и y (промежутки между элементами по горизонтали и вертикали). На рисунке 38 был показан пример использования внешних и внутренних отступов. Например, первая строка сетки имеет верхний внешний отступ mt-3 (margin top 1rem), а третья строка mb-3 (margin bottom 1rem). Центральная строка сочетает в себе отступ сверху и снизу за счет my-5 (margin по оси y между элементами, а так как строка центральная – отступ сверху и снизу).
